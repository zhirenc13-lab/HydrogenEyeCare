# HydrogenEyeCare 1.5 Design

## Goal

HydrogenEyeCare 1.5 improves the existing 1.0 tray app without changing its core 20-minute work and 20-second rest behavior. The update adds reminder-window themes, lightweight daily rest statistics, an enhanced About window, safer log rotation, and a cautious release-size optimization path.

Stability remains more important than file size or new behavior. All changes should stay within the existing .NET 10 + WinForms stack and preserve the tray-first app model.

## Current Baseline

The app currently:

- Starts directly into the Windows system tray.
- Enforces a fixed 20-minute work cycle and 20-second rest reminder.
- Allows a 5-minute delay, up to 2 consecutive delays.
- Provides tray controls for startup, mute, pause/resume, rest now, About, and exit.
- Stores config at `%LOCALAPPDATA%\HydrogenEyeCare\config.json`.
- Writes errors to `%LOCALAPPDATA%\HydrogenEyeCare\error.log`.
- Uses a WinForms reminder window and native tray menu.

## Scope

1.5 includes:

- A tray menu theme selector.
- Three UI themes: Forest Green, Mist Blue, and Rock Gray.
- Reminder-window color theming only; no reminder-window size, layout, or interaction changes.
- Theme-aware styling for the statistics area in the About window.
- In-memory daily successful-rest counting.
- Enhanced About window content.
- `error.log` rotation by size.
- Safe release-size optimization experiments with a fallback path.

1.5 excludes:

- Global theming of all UI.
- Custom drawing for the native tray menu.
- Changes to the fixed 20-minute work, 20-second rest, 5-minute delay, and 2-delay cap.
- Persistent historical statistics.
- NativeAOT.

Light idle detection is optional for 1.5. It should be attempted only after the main 1.5 work is complete and verified. If it proves unstable or time-consuming, it moves to 1.6.

## Theme Design

Add an app theme model with three values:

- `ForestGreen`
- `MistBlue`
- `RockGray`

The default theme is Forest Green because it best matches the eye-care product meaning.

Persist the selected theme in `AppConfig`, so restarting the app keeps the user's last choice. Existing config files without a theme should load safely and fall back to Forest Green.

The tray menu gains a native submenu:

- `界面主题`
  - `森林绿`
  - `迷雾蓝`
  - `暗岩灰`

Only one theme item is checked at a time. Selecting a theme saves config immediately and affects future reminder windows. If a reminder window is currently open, it may either keep its current colors until closed or be recreated only if that is simple and low risk. The lower-risk behavior is acceptable.

## Reminder Window Theming

The reminder window keeps its existing dimensions, layout, no-activation behavior, and interaction model.

The theme changes only:

- Window background color.
- Border color.
- Title text color.
- Countdown number color.
- Prompt text color.
- Delay button background, border, and text color.

The theme palette should be represented in one small local type, such as `ThemePalette`, and passed into `RestReminderForm` when the form is created. This keeps UI coloring separate from timer logic.

## Daily Statistics

Add an in-memory daily stats component that tracks successful rest completions.

Counting rules:

- Completing a full 20-second rest increments today's count by 1.
- Delaying a rest does not increment the count.
- A manually triggered rest from `立即休息` increments the count if the full 20 seconds are completed.
- The count is stored in memory only.
- Exiting the app resets the count.
- Crossing midnight resets the count to 0.

The stats component should own the current date and count. Before reading or incrementing, it checks whether the calendar date has changed and resets if needed.

## About Window

Enhance the About window to show:

- Product name.
- Current app version from assembly metadata.
- GitHub link: `https://github.com/zhirenc13-lab/HydrogenEyeCare`
- Today's successful rest count.
- A short dynamic comment based on today's count.

Suggested comment bands:

- 0: `今天还没有完成远眺，先从下一次提醒开始。`
- 1-3: `今天已经开始照顾眼睛了。`
- 4-7: `今天节奏不错，继续保持。`
- 8 or more: `今天护眼执行得很稳定，注意也要离开座位活动一下。`

The About window should use the current theme only for the statistics/comment area. The rest of the dialog should remain simple and system-like.

## Log Rotation

Before writing an exception to `error.log`, check the current file size.

If `error.log` is larger than 5 MB:

- Clear it before writing the new log entry.
- Keep using the same `error.log` path.
- Do not create archive files.

If rotation fails, the logger should avoid throwing outward. Logging failure must not crash or interrupt the app.

## Light Idle Detection

Light idle detection is second priority for 1.5.

Target behavior:

- Use `GetLastInputInfo` to detect 3 minutes with no keyboard or mouse input.
- When the user has been idle for 3 minutes during a working cycle, suspend work timing.
- When the user returns, reset to a fresh 20-minute work cycle.

This logic must be tested carefully because it touches the timing state machine. If the main 1.5 features are not yet fully verified, or if idle detection creates unclear edge cases, it should be deferred to 1.6.

## Release Size Optimization

The app should not use NativeAOT.

The release process may try trimming as a safe optimization, but the trimming output is acceptable only if manual validation passes. Keep a stable self-contained single-file publish path available as fallback.

Manual validation for a trimmed build must include:

- App starts into the tray.
- Tray menu opens.
- Theme can be changed.
- Immediate rest opens the reminder window.
- Delay works.
- Completing a rest increments stats.
- About window opens and displays correct content.
- App exits and restarts normally.

If the trimmed build breaks any core behavior, release using the stable 1.0-style self-contained single-file configuration.

## Testing Strategy

Implementation should include focused tests where practical:

- Theme config defaults and persistence.
- Daily stats increment and midnight reset.
- Log rotation threshold behavior.

Manual runtime verification is required for WinForms UI behavior:

- Tray startup.
- Theme submenu selection.
- Reminder window colors for all three themes.
- About window content and themed statistics area.
- Rest completion count.
- Log rotation does not interrupt app behavior.
- Publish output smoke test, especially if trimming is enabled.

## Success Criteria

1. A user can choose Forest Green, Mist Blue, or Rock Gray from the tray menu.
2. Reminder windows use the selected theme colors without changing shape or behavior.
3. The About window displays version, GitHub link, today's rest count, and a dynamic comment.
4. Completing a full rest increments today's count; delays do not.
5. Today's count is memory-only and resets across midnight or app restart.
6. `error.log` is cleared before writing when it exceeds 5 MB.
7. Release-size optimization is attempted only in a reversible way, with stable publishing kept as fallback.
8. Light idle detection is either verified as stable or deferred to 1.6.
