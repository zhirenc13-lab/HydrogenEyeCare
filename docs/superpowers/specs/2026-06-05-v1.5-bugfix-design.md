# HydrogenEyeCare 1.5 Bugfix 设计说明

## 目标

本次 bugfix 修复 1.5 验收中发现的三个问题：

- 200% 系统缩放下，提醒窗口和关于窗口存在文字裁切、布局挤压。
- “今日成功远眺次数”应是真正的今日统计，而不是程序本次运行统计。
- 用户手动暂停后，电脑休眠/唤醒不应自动恢复计时。

本次只修复这三个问题，不加入新的功能，不做全局主题化，不改 20 分钟工作、20 秒休息、5 分钟延迟、最多 2 次延迟的核心规则。

## 问题一：200% 缩放下窗口文字裁切

### 现象

在 Windows 系统缩放为 200% 时：

- 关于窗口中 GitHub 链接横向显示不完整。
- 关于窗口中的统计区域和动态评语被垂直裁切。
- 提醒窗口中倒计时数字过大，提示语和延迟按钮被挤压。

### 设计方向

采用组合方案：

- 关于窗口使用弹性布局，并降低内容密度。
- 提醒窗口使用受控弹性布局，只在必要处放宽尺寸、列宽和字号。

### 关于窗口调整

关于窗口应继续保持简洁、系统化风格，但不再依赖固定行高硬塞内容。

调整要求：

- 使用更弹性的 `TableLayoutPanel` 行高。
- 统计区域高度根据内容留足空间，不能裁切动态评语。
- GitHub 不再裸显示完整长 URL，改为链接文本 `GitHub 项目主页`。
- 点击该链接仍打开 `https://github.com/zhirenc13-lab/HydrogenEyeCare`。
- 版本号、今日成功远眺次数、动态评语继续保留。
- 统计区域继续使用当前主题的轻量配色。

验收标准：

- 200% 缩放下，关于窗口内所有文字完整可见。
- GitHub 链接不横向溢出。
- 统计次数和动态评语不被裁切。
- “确定”按钮位置正常，不遮挡内容。

### 提醒窗口调整

提醒窗口是核心打断体验，不做完全自由变形，只做受控弹性。

调整要求：

- 保留无边框、置顶、不抢焦点、右下角弹出的交互形态。
- 允许根据高 DPI 情况放宽窗口尺寸和内部布局空间。
- 倒计时数字不得压住标题、提示语或按钮。
- 提示语和延迟按钮在 200% 缩放下必须完整显示。
- 三套主题颜色继续生效。

验收标准：

- 200% 缩放下，提醒窗口标题、倒计时、提示语、延迟按钮完整显示。
- 延迟按钮文字不裁切。
- 窗口仍保持轻量提醒形态，不变成大弹窗。
- 20 秒倒计时、延迟、完成休息逻辑不变。

## 问题二：今日远眺统计逻辑

### 当前问题

1.5 当前实现中，今日统计只保存在内存里。程序真正退出后再次启动，统计会从 0 开始。

这与用户对“今日成功远眺次数”的理解不一致。既然文案叫“今日”，它就应该代表当天统计，而不是“本次运行”。

另外，曾出现过一次显示 50 次的异常现象。该现象需要通过防重复计数和后续复现来定位，不能只靠改文案掩盖。

### 设计方向

保留“今日成功远眺次数”文案，并把统计改为当天持久化。

调整要求：

- 完整完成一次 20 秒休息后，今日成功远眺次数 +1。
- 点击延迟不计数。
- 程序退出再打开，当天次数仍保留。
- 跨零点后，今日次数清零。
- 统计数据可以保存到现有配置文件或单独 stats 文件；实现时优先选择简单、低风险方案。
- 增加防重复计数保护：同一次休息窗口最多只能计数一次。

验收标准：

- 完整休息一次后，关于窗口显示次数 +1。
- 延迟休息后，次数不增加。
- 退出程序再打开，当天次数仍保留。
- 跨零点后重新读取统计时，次数归零。
- 同一次休息不会因为窗口关闭、计时器重复 Tick 或事件重入而重复 +1。

## 问题三：暂停后休眠/唤醒自动恢复

### 当前问题

当前程序在系统休眠/唤醒、锁屏/解锁后会重置工作周期。若用户先手动点击“暂停计时”，再让电脑休眠，唤醒后程序会回到工作计时状态。

从代码行为看，这是现有系统事件重置逻辑导致的自然结果；但从产品体验看，手动暂停应具有更高优先级。

### 采用方案 B

用户确认采用方案 B：

- 如果用户手动暂停了计时，休眠/唤醒后仍保持暂停。
- 如果用户手动暂停了计时，锁屏/解锁后也应保持暂停。
- 只有用户手动点击“恢复计时”，程序才继续工作周期。

### 设计方向

系统休眠/唤醒、锁屏/解锁仍然可以表示用户离开过电脑；但它不应覆盖用户明确做出的“暂停计时”选择。

调整要求：

- 当状态为 `Paused` 时，系统休眠/唤醒不调用恢复计时。
- 当状态为 `Paused` 时，系统锁屏/解锁不调用恢复计时。
- 非暂停状态下，原有系统休息后重置工作周期的逻辑保持不变。

验收标准：

- 正常工作计时中休眠/唤醒后，仍重置为新的 20 分钟工作周期。
- 手动暂停后休眠/唤醒，醒来后仍显示暂停。
- 手动暂停后锁屏/解锁，回来后仍显示暂停。
- 暂停状态下不会弹出休息提醒。
- 点击“恢复计时”后，程序恢复工作周期。

## 不在本次范围内

本次 bugfix 不处理：

- 轻量挂机检测。
- 发布体积优化。
- NativeAOT。
- 全局主题化。
- 历史统计图表。
- 多日统计记录。
- 新的设置窗口。

## 验证计划

自动化验证：

- 今日统计持久化。
- 跨零点清零。
- 延迟不计数。
- 同一次休息防重复计数。
- 暂停状态下系统重置逻辑不自动恢复。

手动验证：

- Windows 缩放设置为 200%。
- 打开关于窗口，确认文字、统计区、链接和按钮不裁切。
- 触发立即休息，确认提醒窗口在 200% 下完整显示。
- 切换三套主题，确认提醒窗口和关于统计区配色正常。
- 完成一次 20 秒休息，确认今日次数 +1。
- 退出程序再打开，确认今日次数保留。
- 手动暂停后休眠/唤醒，确认仍为暂停状态。
